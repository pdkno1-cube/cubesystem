# PART 1. 시스템 전체 그림

## 핵심 구조 한눈에 보기

```
[CEO] → 말을 던진다
   ↓
[🧭 GUIDE]  ← 모든 대화에서 가장 먼저 작동
   - 의도 파악
   - 어느 팀에 요청할지 판단
   - 바로 쓸 수 있는 지시문 생성
   - CEO의 지시 능력 코칭
   ↓
[👑 PRIME 기반 8개 팀이 실행]
```

# PART 3. 일상 운영 — 상황별 사용법

## 🔵 상황 1: 아이디어가 있는데 어떻게 시작해야 할지 모를 때

**GUIDE에게 그냥 말한다** (어느 터미널이든):
```
[입력 예시]
물류 창고 입출고 관리 시스템 만들고 싶어.
거래처별로 재고 현황 보여주고
입고/출고 처리할 수 있어야 해.
```

**GUIDE가 해주는 것:**
- 어느 팀에 어떤 순서로 요청해야 하는지 알려줌
- 바로 복사해서 쓸 수 있는 지시문 만들어줌
- 더 잘 말하는 방법 코칭해줌

## 🟢 상황 2: 새 프로젝트 시작 (정석 순서)

```
Step 1. T-7 열기 → /design [아이디어]
        → PRD 문서 + 아키텍처 문서 생성

Step 2. T-8 열기 → 설계서 기반 위협 모델링 요청
        → 보안 요구사항 → PRD에 추가

Step 3. T-1 열기 → PRD 기반 티켓 발행 지시
        → TICKET-NNN 파일 생성

Step 4. T-2 + T-3 동시 열기 → 각 티켓 기반 개발 시작

Step 5. T-8 → 개발 완료 후 OWASP 취약점 점검

Step 6. T-4 → 보안 통과 후 테스트 & 배포
```

# PART 4. CEO 지시 능력 키우기

## 좋은 지시문의 7요소

GUIDE가 자동으로 포함시켜주지만, CEO도 익혀두면 좋습니다:

```
1. 목적:     왜 이 작업이 필요한가
2. 대상:     어떤 기능/파일/시스템
3. 요구사항: 구체적으로 무엇을
4. 제약:     지켜야 할 기술/규칙
5. 완료기준: 언제 "완료"인가
6. 참고:     관련 PRD/ARCH/티켓
7. 우선순위: 긴급 여부
```

## 지시 실력 성장 단계

```
Lv.1 초급: "이거 만들어줘"
           → GUIDE가 전부 대신 구조화해줌

Lv.2 중급: 목적 + 요구사항을 스스로 말할 수 있음
           → GUIDE가 제약/완료기준만 추가해줌

Lv.3 고급: 7요소를 갖춘 지시문을 직접 작성
           → GUIDE는 검토·개선만 해줌

Lv.4 전문가: 팀 배정까지 스스로 판단
           → GUIDE 없이도 직접 팀에 지시 가능
```

# PART 5. 명령어 치트시트

```
/guide [요청]      → GUIDE: 분석 + 지시문 생성 + 코칭
/design [아이디어] → T-7: PRD + 아키텍처 설계
/security [대상]   → T-8: 보안 검토
/build [목표]      → 해당팀: Production-ready 코드
/audit             → T-8+T-4: 보안·속도·비용 전체 진단
/scale             → T-3+T-4: 트래픽 대응 리팩토링
/skill [이름]      → T-6: 스킬 검색·개발
/status            → T-5: 전체 현황 대시보드
```

# PART 6. 자주 하는 실수

| 실수 | 올바른 방법 |
|---|---|
| 설계 없이 바로 개발 지시 | T-7 먼저 → `/design` 실행 |
| 보안 검토 없이 배포 | T-8 → OWASP 점검 후 T-4 배포 |
| 어느 팀에 할지 모름 | GUIDE에 그냥 말하면 라우팅해줌 |
| 지시가 너무 막연함 | GUIDE가 7요소 갖춰서 대신 써줌 |
| 두 팀이 같은 일 중복 | T-5에서 충돌 중재 요청 |
