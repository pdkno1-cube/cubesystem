# PRD-MASTEROS-v1: The Master OS -- 1인 100에이전트 자율 경영 시스템

> 작성: TEAM_G (ARCHITECT + PRD_MASTER) | 버전: v1.0 | 2026.02.26

---

## 개요

1인 회장이 100개 AI 에이전트를 지휘하여 무한 확장 가능한 법인 워크스페이스를 운영하는 중앙 집권형 자율 경영 OS.
God Mode 대시보드 위에서 법인 생성, 에이전트 할당, 파이프라인 실행, 과금 통제를 단일 인터페이스로 수행하며, 최종적으로 B2B SaaS 피봇을 목표로 한다.

---

## 배경 & 목적

### 해결하려는 문제

회장은 3PL 물류, 마케팅 에이전시, F&B 프랜차이즈, IT SaaS, 크립토 트레이딩 등 다수 사업을 동시 전개하는 연쇄 창업가다. 인간의 물리적 시간 한계, 파편화된 데이터, 행정 오류(Human Error), 인건비 증가가 스케일업의 병목이다.

### 비전

- **중앙 집권형 자율 주행 엔터프라이즈**: 1명의 인간 지휘관 + 100명의 AI 에이전트가 결합된 조직
- **무한 확장성(Lego-Block Scalability)**: 새 사업 아이디어가 떠오를 때마다 1초 만에 법인 워크스페이스를 생성하고 에이전트를 조립
- **휴먼 에러 0%**: 입찰가 산출, 서류 누락, 세무 정산 등 치명적 영역에 다중 페르소나 + 내부 감사팀(Red Team) 투입
- **B2B SaaS 피봇(End Game)**: 내부 최적화 완료 후 워크스페이스 모듈을 월 구독형 SaaS로 외부 판매

### 왜 지금 해야 하는가

- AI 에이전트 기술(LangGraph, MCP)이 실무 투입 가능 수준에 도달
- 사업 포트폴리오가 확대될수록 수동 관리 비용이 기하급수적으로 증가
- 선점 효과: 1인 다법인 자율 경영 SaaS는 아직 시장에 부재

## 사용자 스토리

> 현재 사용자: 총괄 회장 1인 (The Creator)
> 향후 확장: B2B 고객사 (화주사, 가맹점 등) -- 멀티테넌트 전환

### US-01: 전체 법인 조감

As a 총괄 회장, I want God Mode 대시보드에서 전체 법인의 실시간 현황을 한 눈에 조감, So that 각 법인의 파이프라인 상태, 에이전트 가동률, 크레딧 소모량을 즉시 파악하고 의사결정을 내릴 수 있다.

### US-02: 신규 법인 1초 생성

As a 총괄 회장, I want 대시보드에서 [+ 새 법인 생성] 버튼 하나로 신규 워크스페이스를 즉시 개설, So that 새로운 사업 아이디어를 떠올리자마자 독립 실행 법인을 가동할 수 있다.

### US-03: 에이전트 Drag & Drop 할당

As a 총괄 회장, I want 에이전트 풀에서 원하는 에이전트를 마우스로 끌어다 특정 법인에 자유롭게 할당/회수, So that 사업 상황에 따라 에이전트 리소스를 유연하게 재배치할 수 있다.

### US-04: 파이프라인 실행 및 모니터링

As a 총괄 회장, I want 4대 핵심 파이프라인(정부조달, 서류검증, OSMU 마케팅, 자율유지보수)을 원클릭 실행하고 실시간 진행 상태를 모니터링, So that 실무에 직접 뛰지 않고 최종 승인만으로 사업을 운영할 수 있다.

### US-05: 크레딧 및 과금 통제

As a 총괄 회장, I want 법인별/에이전트별 AI 토큰 소모량(마스터 크레딧)을 실시간 추적하고 한도를 설정, So that API 비용을 통제하고 법인별 수익성을 정확히 파악할 수 있다.

### US-06: 시크릿 볼트 관리

As a 총괄 회장, I want 고객사 ID/PW, API 키 등 민감 정보를 중앙 암호화 금고에서 안전하게 관리, So that 보안 사고 없이 다수 외부 서비스를 자동으로 연동할 수 있다.

### US-07: B2B 테넌트 온보딩 (향후)

As a B2B 고객사 관리자, I want 별도의 워크스페이스를 발급받아 자사 전용 에이전트와 파이프라인을 운용, So that 자체 인력 없이 AI 기반 업무 자동화를 구독형으로 활용할 수 있다.

- [ ] 워크스페이스 간 데이터 완전 격리 보장
- [ ] 워크스페이스 목록 조회 (카드/리스트 뷰 전환)

- [ ] Drag & Drop으로 에이전트를 워크스페이스에 할당 (React Flow 기반)

- [ ] **정부조달 입찰 팩토리**: 수집(FireCrawl) -> 자격 대조(RAG/ChromaDB) -> 다중 페르소나 논리 검증 -> 초안 작성 -> 서류 OCR 검수(PaddleOCR) -> 제출 알림
- [ ] **서류 자동 검증 시스템**: 누락 확인 -> 위변조/날짜 판독(OCR) -> 데이터 양식 검사 -> Google Drive 분류 -> Slack 알림
- [ ] **OSMU 마케팅 스웜**: 아이디어 투입 -> 스크립트 분할 생성(블로그/인스타/뉴스레터/숏폼) -> Figma 연동 비주얼 렌더링
- [ ] **AI 자율 유지보수(Auto-Healing)**: 크롤링 차단/API 에러 감지 -> 예비 API 키 스위칭 -> IP 프록시 우회 -> 핫픽스 적용 -> Slack 보고
- [ ] 파이프라인 실행 상태 대시보드 (대기 / 실행중 / 완료 / 에러)
- [ ] 파이프라인별 실행 이력 및 결과 로그

## 사용자 플로우

### Flow 1: 신규 법인 생성 -> 에이전트 할당 -> 파이프라인 가동

```
1. 회장이 God Mode 대시보드에 로그인 (MFA 인증)
2. [+ 새 법인 생성] 버튼 클릭
3. 법인명, 업종, 설명 입력 -> 확인
4. 시스템이 워크스페이스 레코드 생성 + RLS 정책 자동 적용
   (Optimistic UI: 클릭 즉시 카드 노출, 백그라운드 DB 처리)
5. 워크스페이스 상세 페이지 자동 진입
6. 좌측 에이전트 풀 패널에서 필요한 에이전트를 Drag & Drop으로 할당
   - 예: [기획토론 스웜] + [OSMU 마케팅 스웜] + [내부감사 스웜]
7. 파이프라인 선택 및 실행 버튼 클릭
8. 실행 상태 실시간 모니터링 (WebSocket 기반)
9. 완료 시 결과 리포트 확인 및 최종 승인
```

**Edge Cases:**
- 에이전트 풀에 가용 에이전트가 부족한 경우: 우선순위 기반 자동 재할당 제안
- 워크스페이스 생성 실패 시: 에러 Toast 표시 + 재시도 버튼
- 파이프라인 중간 실패 시: 실패 노드 하이라이트 + 재실행/스킵 옵션

### Flow 2: 정부조달 입찰 자동 수집 -> 검증 -> 제출

```
1. 회장이 [엉클로지텍] 워크스페이스 진입
2. 정부조달 입찰 팩토리 파이프라인 선택
3. 수집 조건 설정 (키워드, 지역, 금액 범위, 마감일 등)
4. [파이프라인 실행] 클릭
5. Stage 1 - 수집: FireCrawl이 나라장터 등 조달 사이트 크롤링
6. Stage 2 - 자격 대조: ChromaDB에 저장된 회사 자격 정보와 RAG 매칭
   -> 적격 입찰 건 필터링
7. Stage 3 - 논리 검증: 다중 페르소나(낙관/비관/현실) 입찰 적합성 토론
   -> 토론 요약 카드 생성
8. Stage 4 - 초안 작성: 입찰 제안서 초안 자동 생성
9. Stage 5 - OCR 검수: PaddleOCR로 첨부 서류 누락/오류 검사
10. Stage 6 - 제출 준비 완료 알림 (Slack + 대시보드)
11. 회장이 최종 검토 후 제출 승인
12. 결과 감사 로그 자동 기록
```

**Edge Cases:**
- 크롤링 차단 시: Auto-Healing이 프록시 우회 자동 시도, 실패 시 수동 개입 알림
- RAG 매칭 신뢰도 낮은 경우: 회장에게 수동 확인 요청 알림
- OCR 판독 불가 서류: 수동 확인 큐에 추가 + Slack 알림

### Flow 3: OSMU 마케팅 아이디어 -> 다중 콘텐츠 생성

```
1. 회장이 대상 워크스페이스 진입 (예: 디어버블)
2. OSMU 마케팅 스웜 파이프라인 선택
3. 마케팅 아이디어/주제 입력 (텍스트 + 참고 이미지 첨부 가능)
   (Upload First: 이미지 첨부 즉시 백그라운드 업로드 시작)
4. 타겟 채널 선택 (블로그 / 인스타그램 / 뉴스레터 / 숏폼 중 다중 선택)
5. [생성 시작] 클릭
6. Stage 1 - 스크립트 분할 생성: 채널별 톤&매너에 맞춘 카피 자동 생성
7. Stage 2 - 비주얼 렌더링: Figma API 연동으로 채널별 이미지/썸네일 생성
8. Stage 3 - 프리뷰: 채널별 결과물 미리보기 카드 UI
9. 회장이 개별 수정 또는 일괄 승인
10. 승인된 콘텐츠 Google Drive 저장 + 배포 큐 등록
11. 실행 결과 리포트 (소요 시간, 크레딧 사용량)
```

**Edge Cases:**
- Figma API 할당량 초과: 대체 이미지 생성 방식 자동 전환 (Dall-E 등)
- 생성 결과 품질 미달 시: 재생성 버튼 (프롬프트 자동 개선 적용)
- 부분 실패 시: 성공 채널만 우선 노출, 실패 채널 재시도 옵션

## 완료 기준 (Acceptance Criteria)

### AC-01: 인증 및 접근 제어

- [ ] 회장 계정으로 이메일 + MFA TOTP 인증 후 로그인 성공
- [ ] 미인증 상태에서 모든 보호 라우트 접근 시 로그인 페이지로 리다이렉트
- [ ] JWT 만료 시 Refresh Token으로 자동 갱신, Refresh Token 만료 시 재로그인 요구

### AC-02: 워크스페이스 관리

- [ ] 신규 워크스페이스 생성 시 1초 이내 UI 반영 (Optimistic UI)
- [ ] 생성된 워크스페이스에 Supabase RLS 정책이 자동 적용됨을 검증 (타 워크스페이스 데이터 접근 불가)
- [ ] 워크스페이스 아카이브 시 하위 에이전트 자동 풀 복귀 확인
- [ ] 10개 이상 워크스페이스 동시 운용 시 대시보드 로딩 2초 이내

### AC-03: 에이전트 풀 및 할당

- [ ] Drag & Drop으로 에이전트를 워크스페이스에 할당 시 500ms 이내 UI 반영
- [ ] 에이전트 상태(미할당/가동중/정지)가 실시간으로 정확히 표시
- [ ] 스웜 템플릿 적용 시 포함된 전체 에이전트가 일괄 할당
- [ ] 이미 할당된 에이전트를 다른 워크스페이스로 이동 시 기존 워크스페이스에서 자동 해제

### AC-04: 파이프라인 실행

- [ ] 4대 파이프라인 각각 실행 버튼 클릭 후 5초 이내 큐 삽입 확인
- [ ] 파이프라인 실행 중 각 Stage 진행 상태가 실시간으로 대시보드에 표시
- [ ] 파이프라인 실패 시 실패 Stage와 에러 메시지가 명확히 표시
- [ ] 파이프라인 완료 시 결과 리포트와 Slack 알림이 동시에 발송

### AC-05: 크레딧 및 과금

- [ ] 에이전트 작업 시 크레딧 소모량이 실시간 차감 반영
- [ ] 법인별 크레딧 한도 도달 시 알림 발송 + 자동 정지 옵션 작동
- [ ] 월별 크레딧 사용량 리포트가 정확히 생성

### AC-06: 시크릿 볼트

- [ ] 시크릿 저장 시 AES-256-GCM으로 암호화되어 DB에 기록됨을 검증
- [ ] 클라이언트 응답에 복호화된 시크릿 값이 절대 노출되지 않음을 검증
- [ ] 시크릿 접근 시 감사 로그에 자동 기록

### AC-07: 감사 로그

- [ ] 모든 CRUD 작업, 파이프라인 실행, 시크릿 접근이 빠짐없이 기록
- [ ] 로그 필터링 (날짜, 액션 유형, 워크스페이스)이 정상 작동
- [ ] 90일 이상 경과 로그의 자동 아카이브 처리

### AC-08: MCP 연동

- [ ] FireCrawl, PaddleOCR, Google Drive, Figma, Slack 각 커넥터가 정상 연결 및 데이터 송수신
- [ ] 커넥터 장애 시 60초 이내 감지 + 30초 이내 Slack 알림 발송
- [ ] Auto-Healing이 예비 API 키 스위칭을 자동 수행

### AC-09: 성능 기준

- [ ] God Mode 대시보드 LCP < 2초 (Lighthouse 측정)
- [ ] API 응답 시간 95th percentile < 500ms
- [ ] 100개 이상 에이전트 동시 운용 시 시스템 안정성 유지
